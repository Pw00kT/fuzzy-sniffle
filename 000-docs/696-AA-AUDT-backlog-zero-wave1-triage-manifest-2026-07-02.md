(manifest 695, PR #939). **Needs-human digest:** issue #941.

## Method

1. Six read-only triage agents classified all 230 beads against the repo (disk,
   git history, merged PRs, live CI/branch-protection state), each verdict carrying
   citable evidence. Verdict split: 57 done-drift, 37 obsolete, 116 still-valid,
   16 needs-human, 4 reserved (the Databricks Part-2b builds held by a concurrent
   session's checkpoint).
2. Three independent skeptic agents re-verified all 94 closable verdicts with
   fresh eyes. **8 were refuted (8.5%)** and stay open — including two cases of
   live spec-vs-validator drift the close would have buried, and a compare-page
   "done" whose acceptance criteria were unmet. Four gold-standard tier beads were
   additionally held as contingent on an open needs-human program decision.
3. Survivors were restructured: every still-valid bead now sits under a
   plain-English epic (six new epics + the existing program epics; the four
   reserved Databricks skill builds became subtree parents for their 33 file-level
   beads). Needs-human items stay open, labeled, each with a question +
   recommended default + decision-by 2026-07-16.
4. Closes below execute one bead per `bd close` with JSONL flush + per-id state
   verification (bd rapid-write race discipline), deepest-first so children close
   before their containers, with a second pass for dependency-refused closes.
   A stratified sample audit follows before this manifest's PR merges.

Already executed ahead of this manifest (Phase C quick wins, merged PR #940,
squash 0f08c52bd): the sentry Overview-gate fix and the HubSpot 10-skill v2.0.0
